# Metric Analysis Skill - 指标查询与分析技能

## 元数据
- **名称**: metric-analysis
- **描述**: 分析各类业务指标数据，支持单指标查询、多地区对比、趋势分析和排名查询
- **版本**: 2.0.0

## 何时使用

当用户需要以下类型的问题时使用此技能：

1. **单指标查询** - "北京招聘数量是多少"
2. **多地区对比** - "对比北京上海广东的招聘数量"
3. **趋势分析** - "北京招聘数量最近6个月趋势"
4. **排名查询** - "招聘数量排名前五的省份"
5. **多维度查询** - "北京本科招聘薪资"

## 五阶段处理流程

```
┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐
│ Stage 0 │ → │ Stage 1 │ → │ Stage 2 │ → │ Stage 3 │ → │ Stage 4 │
│ 意图分类 │    │ 指标识别 │    │ 维度标准化│    │ 数据查询 │    │ 洞察生成 │
└─────────┘    └─────────┘    └─────────┘    └─────────┘    └─────────┘
```

### Stage 0: 意图分类
- 识别闲聊 vs 数据查询
- 闲聊关键词：你好、在吗、你是谁
- 默认进入数据查询流程

### Stage 1: 指标识别（两阶段）

**召回阶段**：
- BM25全文检索（MySQL MATCH AGAINST）
- 向量相似度（text-embedding-v2 + cosine）
- 同义词倒排（HashMap O(1)）

**精排阶段**：
- 大模型语义确认
- 提取维度信息
- 置信度阈值：0.6

### Stage 2: 维度标准化

| 维度 | 处理方式 |
|------|---------|
| 地区 | 名称→编码；级别查询标记 |
| 时间 | latest/last:N/具体日期 |
| 其他 | 默认值填充 |

### Stage 3: 数据查询
- 动态SQL构建
- 参数化查询
- 多数据源支持

### Stage 4: 洞察生成
- 数据解读
- 趋势分析
- 结论建议

## 支持地区

- 具体地区：北京、上海、广东、江苏、浙江、杭州、深圳、广州等
- 级别查询：各省份（省级）、各市（市级）、全国

## 分析流程

### 步骤1: 意图识别
识别用户查询类型：
- `single` - 单指标单地区查询
- `compare` - 多地区对比
- `trend` - 趋势分析
- `ranking` - 排名查询

### 步骤2: 参数提取
提取关键参数：
- `metric` - 指标名称或ID
- `region` - 地区（单个或多个）
- `timeRange` - 时间范围（默认latest）
- `dimensions` - 其他维度（学历、企业类型等）

### 步骤3: 执行查询
根据类型调用相应方法：
- `queryMetricCurrentValue()` - 查询单指标当前值
- `queryMetricComparison()` - 多地区对比
- `queryMetricTrend()` - 趋势分析
- `queryMetricRanking()` - 排名查询

### 步骤4: 结果解读
- 分析数据含义
- 提供同比变化解读
- 给出简要结论或建议

## 输出规范

1. **数值呈现**: 保留2位小数，带单位
2. **同比说明**: 如"同比增长12.5%"
3. **趋势描述**: 上升↗️、下降↘️、平稳➡️
4. **结论建议**: 基于数据的简要分析

## 注意事项

1. 指标名称支持模糊匹配，如"招聘"可匹配"招聘岗位数量"
2. 地区名称支持同义词，如"帝都"="北京"
3. 趋势分析默认返回最近6个月数据
4. 排名查询默认返回TOP20
5. 未指定时间默认使用最新数据
