# 指标分析技能

## 元数据
- **名称**: metric-analysis
- **描述**: 分析各类业务指标数据，支持单指标查询、多地区对比、趋势分析和排名查询
- **作者**: metric-analyst-agent
- **版本**: 1.0.0

## 何时使用

当用户需要以下类型的问题时使用此技能：

1. **单指标查询** - "北京招聘数量是多少"
2. **多地区对比** - "对比北京上海广东的招聘数量"
3. **趋势分析** - "北京招聘数量最近6个月趋势"
4. **排名查询** - "招聘数量排名前五的地区"
5. **多指标分析** - "北京的人才和专利情况"

## 可用指标

| 指标名称 | 说明 | 单位 |
|---------|------|------|
| 招聘数量 | 企业发布的招聘岗位数量 | 个 |
| 专利数量 | 申请的专利总数 | 件 |
| 企业数量 | 注册企业总数 | 家 |
| 投资金额 | 固定资产投资总额 | 亿元 |
| 人才数量 | 高层次人才总数 | 人 |
| 研发费用 | 企业研发投入总额 | 亿元 |

## 支持地区

- 北京、上海、广东、江苏、浙江
- 全国（汇总数据）

## 分析流程

### 步骤1: 意图识别
识别用户查询类型：
- `single` - 单指标单地区查询
- `compare` - 多地区对比
- `trend` - 趋势分析
- `ranking` - 排名查询
- `multi` - 多指标分析

### 步骤2: 参数提取
提取关键参数：
- `metric` - 指标名称
- `region` - 地区（单个或多个）
- `timeRange` - 时间范围（默认最近）
- `topN` - 排名数量（默认5）

### 步骤3: 执行查询
根据类型调用相应工具：
- 使用 `queryMetricCurrentValue` 查询单指标
- 使用 `queryMetricComparison` 进行多地区对比
- 使用 `queryMetricTrend` 查询趋势
- 使用 `queryMetricRanking` 查询排名

### 步骤4: 结果解读
- 分析数据含义
- 提供同比变化解读
- 给出简要结论或建议

## 输出规范

1. **数值呈现**: 保留2位小数，带单位
2. **同比说明**: 如"同比增长12.5%"
3. **趋势描述**: 上升↗️、下降↘️、平稳➡️
4. **结论建议**: 基于数据的简要分析

## 示例

### 示例1: 单指标查询
**用户**: 北京招聘数量是多少？
**响应**: 北京招聘数量为 51,600.50 个（2026年3月），同比增长12.5%，呈现上升趋势↗️

### 示例2: 多地区对比
**用户**: 对比北京上海广东的招聘数量
**响应**:
招聘数量地区对比（2026年3月）：
- 北京: 51,600.50 个 (同比: +12.5%) ↗️
- 上海: 35,135.43 个 (同比: -4.6%) ↘️
- 广东: 29,967.92 个 (同比: -9.2%) ↘️

**结论**: 北京招聘需求最旺盛，上海和广东呈现下降趋势。

### 示例3: 趋势分析
**用户**: 北京招聘数量最近6个月趋势
**响应**:
北京招聘数量近6个月趋势：
- 2025年10月: 54,998.12
- 2025年11月: 43,547.57
- 2025年12月: 44,195.35
- 2026年1月: 43,444.36
- 2026年2月: 55,025.61
- 2026年3月: 51,600.50

**趋势**: 整体波动较大，近期趋于稳定。

## 注意事项

1. 指标名称支持模糊匹配，如"招聘"可匹配"招聘数量"
2. 地区名称需准确，不支持模糊匹配
3. 趋势分析默认返回最近6个月数据
4. 排名查询默认返回TOP5
