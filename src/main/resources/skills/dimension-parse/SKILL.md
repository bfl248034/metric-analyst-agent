# Dimension Parse Skill - 维度解析技能

## 功能说明

从用户查询中解析各类维度的具体值，支持自然语言识别和编码转换。

## 支持维度

### 1. 地区维度 (region)

**编码规则**：6位国标编码
- 100000 = 全国
- 110000 = 北京市
- 310000 = 上海市
- 440000 = 广东省
- 320000 = 江苏省
- 330000 = 浙江省

**级别标记**：
- 省级 = 各省份/各省/不同省份查询
- 市级 = 各市/不同城市查询
- 全国 = 全国汇总数据

**识别示例**：
| 用户表述 | 解析结果 |
|----------|----------|
| 北京/帝都/京城 | 110000 |
| 上海/魔都 | 310000 |
| 全国/中国 | 100000 |
| 各省份/各省 | 省级 |
| 各市/不同城市 | 市级 |

### 2. 时间维度 (time)

**格式支持**：
- latest = 最新数据（默认）
- last:N = 近N期，如 last:6 表示近6个月
- YYYY-MM = 具体月份，如 2024-03

**识别示例**：
| 用户表述 | 解析结果 |
|----------|----------|
| 最新/当前 | latest |
| 最近6个月/近半年 | last:6 |
| 2024年3月 | 2024-03 |
| 上个月 | last:1 |

## 维度标准化流程

```
用户输入
    ↓
提取维度值（自然语言）
    ↓
匹配同义词（synonyms字段）
    ↓
转换为标准编码（value_code）
    ↓
应用默认值（未指定维度）
```

## 分组查询处理

当用户表述包含"各XX"、"不同XX"时：
1. 识别为分组查询
2. 使用 region_level 进行地区级别分组
3. 排除汇总值（TOTAL）
4. 返回分组统计结果
