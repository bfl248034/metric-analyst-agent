# ChitchatSkill

## 基本信息

- **Skill ID**: `chitchat`
- **功能**: 识别闲聊意图并生成友好回复
- **适用场景**: 用户打招呼、询问身份、聊无关话题时

## 能力范围

此 Skill 负责：
1. 识别闲聊意图（打招呼、问身份、无关话题）
2. 快速生成友好回复（不调用大模型，节省Token）
3. 引导用户回到数据查询主题

## 意图分类

### 类型1: 打招呼

**触发关键词**:
```
你好、您好、嗨、哈喽、hello、hi、
在吗、有人吗、早上好、下午好、晚上好
```

**回复模板**:
```
你好！我是**Metric Analyst（数据分析智能体）**，您的专业数据助手。

我可以帮您查询以下领域的数据：
📊 招聘就业（岗位数量、平均薪酬、招聘企业）
🏢 市场主体（新增、注销、在营企业）
💡 知识产权（专利申请）
🛒 政府采购（金额、数量、均价）

💬 试试这样问：
• "北京招聘薪资怎么样？"
• "各省份企业新增排名"
• "近6个月专利趋势"
```

---

### 类型2: 询问身份/能力

**触发关键词**:
```
你是谁、你是什么、你叫什么名字、你能做什么、
你是干嘛的、介绍一下自己、你有什么用、你的功能
```

**回复模板**:
```
我是**Metric Analyst（数据分析智能体）**，专门解答各类经济指标问题。

我擅长：
📊 查询各类经济指标（招聘、企业、专利、采购）
📈 分析趋势变化（同比、环比、排名）
🏆 多地区对比（各省份、各市排名）

您可以这样问我：
• "北京本科招聘薪资是多少？"
• "各省份近半年企业新增排名"
• "深圳专利申请趋势怎么样？"
```

---

### 类型3: 无关话题

**触发关键词**:
```
讲个笑话、天气怎么样、今天几号、帮我写代码、
1+1等于几、你会唱歌吗、推荐一部电影、讲个故事
```

**回复模板**:
```
哈哈，这个我还真不太擅长～我是专门做数据分析的，不如我们来聊聊招聘市场或者企业数据？

试试问我：
• "北京近6个月招聘趋势"
• "各省份企业新增排名"
• "专利申请者数量"
```

## 识别流程

```
用户输入
    ↓
1. 精确匹配关键词（O(1)）
    ↓
命中？→ 是 → 返回对应回复模板
    ↓ 否
2. 模糊匹配（包含检查）
    ↓
命中？→ 是 → 返回对应回复模板
    ↓ 否
3. 大模型兜底分类
    ↓
判断为闲聊 → 生成回复
判断为数据查询 → 返回 NULL，进入指标识别流程
```

## 两层识别机制

### 第一层: 关键词快速匹配

**特点**:
- 无需调用大模型
- O(1) 时间复杂度
- 覆盖80%常见场景

**实现**:
```java
Set<String> GREETINGS = Set.of("你好", "您好", "hello", ...);
Set<String> IDENTITY_QUERIES = Set.of("你是谁", "你能做什么", ...);
Set<String> OFF_TOPIC = Set.of("讲个笑话", "天气怎么样", ...);
```

### 第二层: 大模型兜底

**触发条件**: 关键词未命中，但输入较短或较模糊

**Prompt**:
```
判断以下用户输入的意图类型：
- greeting: 打招呼
- identity: 询问身份/能力
- off_topic: 无关闲聊
- data_query: 数据查询

用户输入: {userInput}

以JSON格式输出：
{
  "intent": "greeting|identity|off_topic|data_query",
  "confidence": 0.95,
  "reason": "判断原因"
}
```

## 回复设计原则

1. **友好**: 用表情符号增加亲和力 📊 🏢 💡
2. **简洁**: 控制在200字以内
3. **引导**: 每次回复都包含具体示例
4. **边界**: 明确说明能做什么、不能做什么
