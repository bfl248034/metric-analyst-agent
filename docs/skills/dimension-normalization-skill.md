# DimensionNormalizationSkill

## 基本信息

- **Skill ID**: `dimension_normalization`
- **功能**: 将用户输入的维度值标准化为系统可识别的编码
- **适用场景**: 识别阶段提取的原始维度需要转换为查询参数时使用

## 能力范围

此 Skill 负责：
1. 地区名称标准化（北京 → 110000）
2. 时间表达式计算（近6个月 → [2023-09-30, ..., 2024-02-29]）
3. 其他维度值映射（本科 → RAE_EDU_6）
4. 默认值填充（未指定维度使用默认值）

## 可用工具

### normalizeDimensions

**描述**: 标准化所有维度值

**参数**:
| 参数名 | 类型 | 必填 | 描述 |
|--------|------|------|------|
| indicatorId | string | ✅ | 指标编码 |
| rawDimensions | map | ✅ | 识别阶段提取的原始维度 |

**返回结果**:
```json
{
  "indicatorId": "I_RPA_ICN_RAE_SALARY_AMOUNT",
  "tableId": "ads_rpa_w_icn_recruit_salary_amount_m",
  "explicitDimensions": {
    "region": {"valueCode": "110000", "valueName": "北京市"},
    "education": {"valueCode": "RAE_EDU_6", "valueName": "本科"}
  },
  "implicitDimensions": {
    "data_attr": {"valueCode": "DAT_1", "valueName": "当期"},
    "economic_type": {"valueCode": "TOTAL", "valueName": "全部"}
  }
}
```

### expandTimePeriods

**描述**: 展开时间表达式为具体日期列表

**参数**:
| 参数名 | 类型 | 必填 | 描述 |
|--------|------|------|------|
| timeExpression | string | ✅ | 时间表达式，如 "last:6" |
| frequency | string | ✅ | 指标频率 M(月)/Q(季)/Y(年) |

**时间计算规则**:

#### 月度数据（M）
```
近6个月 → 取每月最后一天：
- 2024-02-29（2月最后一天）
- 2024-01-31（1月最后一天）
- 2023-12-31（12月最后一天）
- 2023-11-30（11月最后一天）
- 2023-10-31（10月最后一天）
- 2023-09-30（9月最后一天）
```

#### 季度数据（Q）
```
近4个季度 → 取每季度末：
- 2024-03-31（Q1末）
- 2023-12-31（Q4末）
- 2023-09-30（Q3末）
- 2023-06-30（Q2末）
```

#### 年度数据（Y）
```
近3年 → 取每年末：
- 2023-12-31
- 2022-12-31
- 2021-12-31
```

## 地区标准化规则

### 级别查询识别

当用户输入包含以下关键词时，识别为级别查询：

| 关键词 | 解析结果 | 说明 |
|--------|----------|------|
| "各省份"、"各省"、"不同省份" | `region_level: "省级"` | 查询所有省级数据 |
| "各市"、"各城市"、"不同城市" | `region_level: "市级"` | 查询所有市级数据 |
| "全国" | `region_id: "100000"` | 查询全国汇总数据 |

### 具体地区识别

支持多种识别方式：

| 输入 | 标准化结果 | 识别方式 |
|------|------------|----------|
| "北京"、"北京市" | `110000` | 直接匹配 |
| "帝都" | `110000` | 同义词匹配 |
| "京城" | `110000` | 同义词匹配 |
| "上海"、"魔都"、"沪" | `310000` | 同义词匹配 |
| "广州"、"羊城"、"穗" | `440100` | 同义词匹配 |
| "深圳"、"鹏城" | `440300` | 同义词匹配 |

### 地区数据结构

```
100000（全国）
├── 110000（北京）省级
├── 310000（上海）省级
├── 440000（广东）省级
│   ├── 440100（广州）市级
│   └── 440300（深圳）市级
└── 330000（浙江）省级
    └── 330100（杭州）市级
```

## 其他维度标准化

### 学历维度

| 输入 | 编码 | 说明 |
|------|------|------|
| 未指定 | `TOTAL` | 全部学历（默认） |
| 大专 | `RAE_EDU_5` | 大学专科 |
| 本科 | `RAE_EDU_6` | 大学本科 |
| 硕士 | `RAE_EDU_7` | 硕士研究生 |
| 博士 | `RAE_EDU_8` | 博士研究生 |

### 数据属性维度

| 输入 | 编码 | 说明 |
|------|------|------|
| 未指定 | `DAT_1` | 当期值（默认） |
| 当期 | `DAT_1` | 当期值 |
| 累计 | `DAT_2` | 累计值 |

### 经济类型维度

| 输入 | 编码 | 说明 |
|------|------|------|
| 未指定 | `TOTAL` | 全部类型（默认） |
| 国有 | `MKE_ECO_1` | 国有经济 |
| 集体 | `MKE_ECO_2` | 集体经济 |
| 私营 | `MKE_ECO_3` | 私营经济 |

## 默认值配置

各维度默认值（当用户未指定时自动填充）：

| 维度 | 默认值 | 适用指标 |
|------|--------|----------|
| region | `100000`（全国） | 所有指标 |
| education | `TOTAL`（全部） | 招聘类指标 |
| economic_type | `TOTAL`（全部） | 企业类指标 |
| data_attr | `DAT_1`（当期） | 所有指标 |
| patent_type | `TOTAL`（全部） | 专利类指标 |

## 标准化流程

```
原始维度（识别阶段输出）
    ↓
1. 遍历每个维度定义
    ↓
2. 用户是否指定了该维度？
   ├── 是 → 标准化为编码值（存入 explicitDimensions）
   └── 否 → 使用默认值（存入 implicitDimensions）
    ↓
3. 特殊处理：
   - 地区：检查是否为级别查询（各省份/各市）
   - 时间：展开为具体日期列表
    ↓
4. 返回标准化后的维度集合
```

## 超范围处理

当用户指定的时间超出数据范围时：

```
用户: "2025年招聘薪资"
系统: "2025年数据尚未发布，为您展示最新数据（2024年2月）"

处理：自动降级为 latest
```
